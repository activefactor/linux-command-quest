# Linux Command Quest 要件定義書

## 1. システム名

Linux Command Quest

## 2. システム概要

ブラウザ上で動作する、Linuxコマンド学習用のミッション型コード入力ゲームを開発する。

学習者は仮想ターミナルにコマンドを入力し、仮想ファイルシステム上でディレクトリ移動、ファイル確認、コピー、権限変更などの操作を行う。システムは入力されたコマンドを解析し、実行結果を表示し、ミッション達成条件を判定する。

実際のOSコマンドは実行せず、すべてブラウザ内の安全な仮想環境で処理する。

## 3. 対象範囲

### 3.1 MVPで作る範囲

- ブラウザで動く1画面アプリ
- CLI風トップページ
- 自動タイピングによる起動演出
- ASCIIアートのタイトル表示
- コマンド入力によるメニュー選択
- `vi` 風の閲覧専用マニュアル画面
- ゲーム中に `exit` でトップページへ戻る機能
- 進捗を確認する専用ページ
- コマンド暗記用のタイピング練習モード
- 20問の本番タイムアタックモード
- Canvasによるターミナル風表示
- コマンド入力欄
- 仮想ファイルシステム
- 基本コマンドの実行
- ミッション1本から3本
- ヒント表示
- 成功、失敗、警告のフィードバック
- 簡易スコア

### 3.2 MVPでは作らない範囲

- 実Linux環境への接続
- ユーザー登録
- 学習履歴の永続保存
- 管理者画面
- 複数人ランキング
- 本物のシェル実行
- ネットワーク通信を伴う演習

## 4. ユーザー種別

| ユーザー | 説明 |
| --- | --- |
| 学習者 | コマンドを学ぶ利用者 |
| 研修担当者 | 将来的にミッションを作成、配布する人 |
| 管理者 | 将来的にユーザーや進捗を管理する人 |

MVPでは学習者のみを対象とする。

## 5. 機能要件

### 5.1 ターミナル表示機能

#### 概要

Canvas上にターミナル風の画面を表示する。

#### 要件

- コマンドプロンプトを表示できる
- 入力済みコマンドを履歴として表示できる
- コマンド実行結果を表示できる
- エラー、警告、成功メッセージを色分けできる
- `clear` 実行時に表示履歴を消去できる
- ミッション進行に応じてシステム通知を表示できる

### 5.2 コマンド入力機能

#### 概要

学習者がコマンドを入力し、Enterで実行できる。

#### 要件

- 入力欄にコマンドを入力できる
- Enterでコマンドを実行できる
- 上下キーで履歴をたどれる
- 空入力は無視する
- 未対応コマンドにはエラーメッセージを返す
- 入力後、コマンドパーサーに文字列を渡す

### 5.3 コマンドパーサー機能

#### 概要

入力された文字列をコマンド名と引数に分解し、対応する処理を呼び出す。

#### MVP対応コマンド

| コマンド | 用途 |
| --- | --- |
| `pwd` | 現在のディレクトリを表示する |
| `ls` | ファイル一覧を表示する |
| `cd` | ディレクトリを移動する |
| `cat` | ファイルの内容を表示する |
| `cp` | ファイルをコピーする |
| `chmod` | ファイル権限を変更する |
| `vi` | マニュアルやファイルを閲覧する |
| `clear` | ターミナル表示を消去する |
| `help` | 使用可能コマンドを表示する |

#### 将来対応コマンド

- `mkdir`
- `touch`
- `mv`
- `rm`
- `grep`
- `find`
- `tail`
- `head`
- `wc`
- `ps`
- `df`
- `du`
- `systemctl`
- `journalctl`
- `curl`

### 5.4 仮想ファイルシステム機能

#### 概要

ブラウザ内にLinux風のディレクトリとファイルを保持する。

#### 要件

- ディレクトリ階層を表現できる
- ファイル名、内容、権限、所有者、種類を保持できる
- 現在の作業ディレクトリを保持できる
- 絶対パスと相対パスを解決できる
- `.` と `..` を解釈できる
- 存在しないパスに対して適切なエラーを返す
- ファイル操作後に状態を更新できる

#### 初期ファイル構成例

```text
/
  home/
    trainee/
      projects/
        readme.txt
        app.conf
        deploy.sh
      logs/
        access.log
        error.log
  etc/
    myapp/
      app.conf
  var/
    log/
      syslog
      nginx/
        access.log
        error.log
```

### 5.5 ミッション管理機能

#### 概要

ミッションの説明、達成条件、ヒント、失敗条件を管理する。

#### 要件

- 現在のミッションを表示できる
- ミッションごとに開始状態を設定できる
- コマンド実行後に達成条件を判定できる
- 達成時に成功メッセージを表示できる
- 未達成時に次の行動を促せる
- 危険操作時に警告またはインシデントを発生させる
- ミッション完了後に解説を表示できる

#### ミッションデータ例

```json
{
  "id": "backup-config-001",
  "title": "設定ファイルをバックアップせよ",
  "description": "/etc/myapp にある app.conf を app.conf.bak としてコピーしてください。",
  "startPath": "/home/trainee",
  "goals": [
    { "type": "fileExists", "path": "/etc/myapp/app.conf.bak" },
    { "type": "fileExists", "path": "/etc/myapp/app.conf" }
  ],
  "hints": [
    "まず cd で /etc/myapp に移動しましょう。",
    "ファイルのコピーには cp を使います。",
    "cp app.conf app.conf.bak を試してみましょう。"
  ],
  "dangerRules": [
    { "command": "mv app.conf app.conf.bak", "level": "warning" },
    { "command": "chmod 777 app.conf", "level": "incident" }
  ]
}
```

### 5.6 ヒント機能

#### 概要

詰まった学習者に段階的なヒントを表示する。

#### 要件

- ヒントボタンを押すと次のヒントを表示する
- 間違いが連続した場合、自動でヒント提案を表示する
- ヒント使用回数を記録する
- ヒント使用時にスコアを減点できる
- 答えを直接出すヒントは最後の段階にする

### 5.7 インシデント演出機能

#### 概要

危険なコマンドや誤操作を、ゲーム内インシデントとして表現する。

#### 要件

- 危険操作を検知できる
- 警告レベルと重大インシデントレベルを分けられる
- インシデントメーターを表示できる
- 重大操作時に画面演出を出せる
- クリア後に、何が危険だったかを解説できる

#### 危険操作例

| 操作 | レベル | 説明 |
| --- | --- | --- |
| `chmod 777 app.conf` | warning | 設定ファイルが誰でも書き込める |
| `chmod -R 777 /` | incident | システム全体の権限事故 |
| `rm app.conf` | warning | 設定ファイル削除 |
| `rm -rf /` | incident | システム破壊操作 |
| バックアップ前の設定変更 | warning | 復旧手段がない |

### 5.8 スコア機能

#### 概要

学習者の操作を評価し、学習の振り返りに使う。

#### 要件

- ミッション開始時に基礎点を設定する
- 正しい確認操作で加点する
- ヒント使用で減点する
- 危険操作で減点する
- 重大インシデントで大きく減点する
- クリア時にランクを表示する

#### 評価軸

- 正確性
- 安全性
- 確認行動
- ヒント依存度
- コマンド数

### 5.9 コマンドタイピング練習機能

#### 概要

Linuxコマンドを覚えるため、意味とコマンドを同時に表示し、学習者が表示されたコマンドを正確にタイピングする練習モードを提供する。

ミッション型ゲームに入る前、またはミッションの合間に、コマンドの意味と入力パターンを短時間で反復練習できるようにする。

#### 要件

- トップページから `typing` または `6` で開始できる
- コマンドの意味を表示する
- 入力すべきコマンドを表示する
- 学習者は表示されたコマンドを入力する
- 入力が完全一致したら正解とする
- 正解後、自動またはEnterで次の問題へ進む
- タイピング練習ではコマンド名だけを対象にし、オプションやパラメータは省略する
- 説明ウィンドウに「※これはコマンドを覚えるゲームなのでオプションやパラメータは、省略しています」と表示する
- 間違えた場合は、正解コマンドの先頭から1文字ずつヒントを表示する
- 練習中はタイムよりも正確さを優先する
- 練習対象は基本コマンドから始める
- `exit` でトップページへ戻れる

#### 練習問題例

| 意味 | 入力するコマンド |
| --- | --- |
| 現在のディレクトリを表示する | `pwd` |
| ファイル一覧を表示する | `ls` |
| ディレクトリを移動する | `cd` |
| ファイルの内容を表示する | `cat` |
| ファイルをコピーする | `cp` |
| ファイル権限を変更する | `chmod` |
| ファイルをエディタで開く | `vi` |

### 5.10 本番タイムアタック機能

#### 概要

タイピング練習モードで学んだコマンドを対象に、意味だけを表示し、対応するコマンドを素早く入力する本番モードを提供する。

20問をランダムに出題し、正確さとスピードを競う。Enterで確定した不正解にはペナルティ秒を加算する。

#### 要件

- トップページから `challenge` または `7` で開始できる
- 1回の本番は20問とする
- 練習モードと同じ問題セットからランダムに出題する
- 本番では意味のみを表示する
- 本番では原則として正解コマンドを表示しない
- 学習者は意味に対応するコマンドを入力する
- 本番でもコマンド名だけを対象にし、オプションやパラメータは省略する
- 正解したら次の問題へ進む
- Enterで回答を確定した時点で不正解だった場合、1回につき1秒をペナルティとして加算する
- 不正解時は、正解コマンドの先頭から1文字ずつヒントを表示する
- キー入力単位のミスタイピング判定はMVPでは行わない
- 本番中は経過時間を表示する
- 結果画面で素の経過時間、ペナルティ秒、最終タイムを表示する
- 結果画面で正答数、誤答数、正確率を表示する
- `exit` で本番を中断してトップページへ戻れる

#### 採点ルール

```text
最終タイム = 経過時間 + ペナルティ秒
ペナルティ秒 = Enter時の不正解回数 * 1秒
正確率 = 正答数 / 入力回数
```

#### ランク例

| ランク | 条件 |
| --- | --- |
| S | 最終タイム60秒以内、正確率95%以上 |
| A | 最終タイム90秒以内、正確率90%以上 |
| B | 最終タイム120秒以内、正確率80%以上 |
| C | それ以外 |

## 6. 非機能要件

### 6.1 安全性

- 実際のOSコマンドは実行しない
- ユーザー入力をシェルに渡さない
- ファイル操作は仮想ファイルシステム内に限定する
- 危険コマンドもシミュレーションとして処理する

### 6.2 操作性

- Enterで実行できる
- ターミナルらしい見た目にする
- 初心者でも次に何をすればよいか分かる表示にする
- ヒントをいつでも確認できる
- ミスしても復帰できる

### 6.3 学習しやすさ

- 1ミッションは3分から10分程度で完了できる
- コマンド実行結果を実際のLinuxに近づける
- エラー文は英語のLinux風メッセージと日本語補足を併記できる
- クリア後に短い解説を表示する

### 6.4 拡張性

- コマンド追加がしやすい構成にする
- ミッションデータをJSONで追加できるようにする
- 仮想ファイルシステムの初期状態をミッションごとに切り替えられるようにする
- 将来的にReactやTypeScriptへ拡張しやすい構成にする

### 6.5 パフォーマンス

- MVPではローカルブラウザで即時動作する
- コマンド実行結果は100ms以内を目標にする
- Canvas描画は60fpsを目標にする
- 大量ログ演出時も入力遅延が出ないようにする

## 7. 画面要件

### 7.1 デザイン方針

#### 概要

インターフェイスは、本物のLinux画面を触っている感覚を重視する。学習用の問題文、ヒント、解説はLinux画面の一部ではなく、ゲーム側が重ねて表示している別レイヤーとして表現する。

#### 要件

- Linux画面は黒背景のCLIを基本にする
- 等幅フォントを使用する
- プロンプト、コマンド、標準出力、エラーをLinux風に表示する
- 学習UIはLinux画面の上に浮いている半透明パネルとして表示する
- 学習UIは背後のLinux画面が少しだけ感じられる透明度にする
- 背後の文字が読めすぎて学習UIの可読性を損なわないよう、軽いブラーを入れる
- 学習UIはターミナルの文字表示と混同しない配色、余白、境界線にする
- 問題文、ヒント、達成条件、解説は学習UI側に表示する
- コマンド実行結果はLinux画面側に表示する
- 警告やインシデントは、Linux画面内のメッセージと学習UI側のアラートを連動させる
- 通常のWebサイト風のボタン中心UIではなく、CLI操作を主役にする

#### 推奨トーン

- Linux画面: 実機感、緊張感、無駄の少ない表示
- 学習UI: 読みやすい、安心感がある、補助教材であることが分かる
- トップページ: CLIでシステムが起動する高揚感
- インシデント画面: アラート、ログ、監視通知に近い臨場感

### 7.2 トップページ

#### 概要

トップページは、一般的なランディングページではなく、CLI起動画面として表示する。

#### 要件

- 初回表示時に黒背景のCLI画面を表示する
- コマンドが自動でタイピングされる演出を表示する
- 起動ログ風のテキストを表示する
- ゲームタイトルをASCIIアートで表示する
- メニュー項目をテキストで表示する
- メニュー選択はコマンド入力で行う
- `start` または `1` でゲームを開始する
- `manual` または `2` で遊び方と世界観を表示する
- `credits` または `3` でクレジットを表示する
- `progress` または `4` で進捗ページを表示する
- 中断中のゲームがある場合、`resume` または `5` でゲームへ戻れる
- `typing` または `6` でタイピング練習モードを開始する
- `challenge` または `7` で本番タイムアタックモードを開始する
- 未定義の入力には、使用可能なコマンドを案内する

#### 起動演出例

```text
$ ssh trainee@linux-command-quest.local
Connecting...
Welcome to Linux Command Quest

$ ./start-training.sh
Loading virtual filesystem...
Loading missions...
Starting terminal trainer...
```

#### メニュー例

```text
[1] start     ゲームを始める
[2] manual    遊び方と世界観を読む
[3] credits   クレジットを見る
[4] progress  進捗を見る
[5] resume    中断したゲームへ戻る
[6] typing    コマンドタイピング練習
[7] challenge 本番タイムアタック

$ 
```

### 7.3 ASCIIアート表示

#### 概要

タイトル、説明画面の見出し、ステージ開始時の演出にASCIIアートを使用する。

#### 要件

- タイトルはトップページで大きく表示する
- 説明画面の見出しにもASCIIアートを使用する
- ASCIIアートは等幅フォントで崩れないようにする
- 小さい画面では横幅に収まる短縮版タイトルを使用できる
- ASCIIアートの下に通常テキストの説明を置く

### 7.4 マニュアル画面

#### 概要

遊び方や世界観は、ゲーム内ファイルを `vi` コマンドで開く形で表示する。

#### 要件

- トップページで `manual` を入力すると、内部的に `vi manual.txt` を実行する演出にできる
- 学習者が直接 `vi manual.txt` と入力してもマニュアルを開ける
- マニュアル画面はLinux内でファイルを開いているように表示する
- 見出しはASCIIアートにする
- 本文は読みやすい日本語テキストにする
- 説明は番号付きで表示する
- マニュアル末尾に戻り方を明記する
- `:q!` または `:wq` でトップページへ戻る
- 入力欄の先頭に `:` は自動表示しない
- 学習者が自分で `:q!` または `:wq` を入力する
- インサートモードは実装しない
- 編集は不可とする
- 入力欄では `:q!` と `:wq` を終了コマンドとして受け付ける
- `i`、`a`、`o` などの編集操作を入力した場合は「閲覧専用です」と表示する

#### マニュアル本文に含める内容

- ゲームの目的
- コマンド入力の基本
- `pwd`、`ls`、`cd` の意味
- ミッションの進め方
- ヒントの使い方
- 危険操作とインシデントの考え方
- `vi` からの終了方法

### 7.5 ゲームプレイ画面

#### メイン画面

##### 表示要素

- ステージ名
- ミッションタイトル
- ミッション説明
- ヒントボタン
- スコア
- インシデントメーター
- Canvasターミナル
- コマンド入力欄
- クリア後の解説モーダル

#### ゲーム制御コマンド

- `exit` で現在のゲームセッションを保持したままトップページへ戻る
- `progress` で進捗ページを表示する
- `resume` でトップページから中断中のゲームへ戻る
- `restart` でゲームを最初から開始する
- `next` でクリア済みミッションから次のミッションへ進む

### 7.6 進捗ページ

#### 概要

現在のゲーム進行状況を、Linux画面とは別のCLI風ページとして確認できる。

#### 要件

- `progress` または `4` で表示できる
- ゲーム中に `progress` を入力して表示できる
- トップページからも表示できる
- 現在のミッションを表示する
- 完了済みミッション数を表示する
- スコアを表示する
- リスク状態を表示する
- 現在の作業ディレクトリを表示する
- `exit` または `back` で前の画面へ戻る
- `start` で新しいゲームを開始できる

### 7.7 タイピング練習画面

#### 概要

コマンドの意味と正解コマンドを同時に表示し、学習者がそのコマンドをタイピングする画面。

#### 表示要素

- 練習モード名
- 現在の問題番号
- コマンドの意味
- 入力すべき正解コマンド
- 入力欄
- 正誤フィードバック
- 正解数
- ミスタイピング数
- `exit` の案内

#### 要件

- 意味とコマンドを同じ画面に表示する
- コマンドは等幅フォントで表示する
- 入力中に一致している部分と間違っている部分を視覚的に区別できる
- 完全一致で正解扱いにする
- 練習ではタイマー表示は任意とする
- `exit` でトップページへ戻る

### 7.8 本番タイムアタック画面

#### 概要

コマンドの意味だけを表示し、対応するコマンドを素早く入力する本番画面。

#### 表示要素

- 本番モード名
- 現在の問題番号
- 全20問中の進捗
- コマンドの意味

## 10. コマンド仕様

### 10.1 `pwd`

- 引数なし
- 現在の作業ディレクトリを表示する

### 10.2 `ls`

- 引数なしの場合、現在のディレクトリを表示する
- `ls path` で指定パスの一覧を表示する
- MVPでは `-l` のみ対応候補とする

### 10.3 `cd`

- `cd path` で作業ディレクトリを変更する
- `cd ..` に対応する
- 存在しないパスではエラーにする
- ファイルを指定した場合はエラーにする

### 10.4 `cat`

- `cat file` でファイル内容を表示する
- ディレクトリを指定した場合はエラーにする
- 存在しないファイルではエラーにする

### 10.5 `cp`

- `cp source dest` に対応する
- MVPではファイルコピーのみ対応する
- コピー先が存在する場合の上書きは警告表示候補とする

### 10.6 `chmod`

- `chmod mode file` に対応する
- MVPでは3桁数値モードのみ対応する
- `777` は警告対象にする
- `-R` はMVPでは未対応または重大警告扱いにする

### 10.7 `vi`

- MVPでは閲覧専用コマンドとして扱う
- `vi manual.txt` でマニュアル画面を開く
- `vi filename` で対応ファイルを閲覧できる
- インサートモードは実装しない
- ファイル編集はできない
- 閲覧画面の入力欄には `:` を事前表示しない
- 学習者が `:q!` または `:wq` を入力する
- `:q!` で閲覧画面を閉じる
- `:wq` でも閲覧画面を閉じる
- `i`、`a`、`o` などを入力した場合は閲覧専用であることを表示する
- 存在しないファイルを指定した場合はエラーにする

### 10.8 ゲーム制御コマンド

Linuxコマンドではなく、ゲーム進行を制御するコマンドとして扱う。

- `exit`: ゲーム中にトップページへ戻る
- `progress`: 進捗ページを表示する
- `resume`: 中断中のゲームへ戻る
- `next`: 次のミッションへ進む
- `restart`: ゲームを最初から開始する

## 11. エラー表示方針

Linuxらしい英語メッセージを基本にしつつ、初心者向けに日本語補足を表示する。

例:

```text
cd: /home/foo: No such file or directory
ヒント: 指定したディレクトリが存在しません。ls で候補を確認してみましょう。
```

## 12. 受け入れ条件

MVPは以下を満たしたら完了とする。

- ブラウザでゲーム画面が表示される
- コマンド入力ができる
- `pwd`、`ls`、`cd`、`cat`、`cp`、`chmod` が動作する
- トップページでCLI風の起動演出が表示される
- トップページでASCIIアートのタイトルが表示される
- `start` または `1` でゲームを開始できる
- `manual`、`2`、または `vi manual.txt` で説明画面を開ける
- 説明画面は `:q!` または `:wq` で閉じられる
- 説明画面の入力欄に `:` が事前表示されない
- ゲーム中に `exit` でトップページへ戻れる
- トップページから `resume` で中断中のゲームへ戻れる
- `progress` で進捗ページを表示できる
- 進捗ページは `exit` または `back` で閉じられる
- `typing` または `6` でタイピング練習モードを開始できる
- タイピング練習モードでは意味とコマンドが同時に表示される
- `challenge` または `7` で20問の本番タイムアタックを開始できる
- 本番タイムアタックでは意味のみが表示される
- 本番タイムアタックではEnter時の不正解ごとに1秒ペナルティが加算される
- 本番タイムアタックの結果画面で最終タイムと正確率が表示される
- 仮想ファイルシステムの状態がコマンドにより変化する
- 少なくとも3つのミッションを順番に遊べる
- ミッション達成時にクリア表示が出る
- 危険操作時に警告が出る
- ヒントを表示できる
- 実OSに影響する処理が存在しない
