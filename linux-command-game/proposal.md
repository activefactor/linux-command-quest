# Linux Command Quest 提案書

## 1. 企画概要

「Linux Command Quest」は、インフラエンジニアの基礎となるコンソール操作を、ミッション形式で体験しながら覚えるコード入力ゲームです。

単にコマンドを暗記するのではなく、ターミナルを起動し、ディレクトリを移動し、ファイルを確認し、権限を変更し、ログを読み、障害対応をするという一連の流れを、ゲームとして安全に練習できることを目指します。

最初は `pwd`、`ls`、`cd`、`cat` などの基本コマンドに慣れることを中心にし、後半ではファイル操作、権限、プロセス、ログ調査、設定変更、障害復旧といった実務に近いミッションへ進みます。

## 2. 目的

- コンソールに触れたことがない人が、ターミナル操作の意味を体験的に理解できるようにする
- Linuxコマンドを、入力・実行・結果確認の流れで覚えられるようにする
- インフラ作業における「確認してから実行する」習慣を身につける
- 権限変更、削除、設定変更など、誤操作が事故につながる作業の怖さを安全に学べるようにする
- 研修、オンボーディング、自習教材として使える形にする

## 3. 想定ユーザー

- インフラエンジニアを目指す初学者
- Webアプリ開発者だがLinux操作に不安がある人
- 新卒・未経験エンジニア向け研修の受講者
- クラウド、サーバー、CI/CD、ログ調査をこれから学ぶ人

## 4. 学習体験のコンセプト

### 4.1 最初は「打つこと」に慣れる

序盤は、正しいコマンドを見ながら入力するタイピングゲームに近い体験にします。

例:

- ターミナルを起動する
- `pwd` で現在地を確認する
- `ls` でファイル一覧を見る
- `cd logs` でディレクトリを移動する
- `cat access.log` でファイルの中身を見る

この段階では、失敗しても大きなペナルティはなく、コマンドの意味、引数、実行結果の見方に慣れることを優先します。

### 4.2 タイピング練習で意味とコマンドを結びつける

基本コマンドを覚えるためのタイピング練習モードを用意します。

練習モードでは、コマンドの意味と正解コマンドを同時に表示します。学習者は表示されたコマンドをそのままタイピングすることで、「この操作にはこのコマンドを使う」という対応関係を少しずつ覚えます。

ゲーム性を保つため、このモードではディレクトリ名、ファイル名、オプションの暗記は求めません。説明ウィンドウには「※これはコマンドを覚えるゲームなのでオプションやパラメータは、省略しています」と表示します。

例:

| 意味 | コマンド |
| --- | --- |
| 現在のディレクトリを表示する | `pwd` |
| ファイル一覧を表示する | `ls` |
| ディレクトリを移動する | `cd` |
| ファイルの内容を表示する | `cat` |
| ファイルをコピーする | `cp` |
| ファイル権限を変更する | `chmod` |

練習では速さよりも、意味を見て、コマンドを正確に打つことを重視します。

### 4.3 本番タイムアタックで素早く思い出す

本番モードでは、練習モードで扱ったコマンドを対象に、意味だけを表示します。学習者は対応するコマンドを思い出して入力します。

1回の本番は20問とし、ランダムに出題します。ただし、同じ本番セッション内で同じ問題は出しません。正確さとスピードの両方を評価するため、基本はタイムアタック形式にします。Enterで回答を確定した時点で不正解だった場合は、1回につき1秒を最終タイムに加算します。

評価例:

```text
最終タイム = 経過時間 + Enter時の不正解回数 * 1秒
```

このモードにより、単なる暗記ではなく「意味を見てすぐコマンドを出せる」状態を目指します。

不正解時は、いきなり答えを出さず、正解コマンドの先頭から1文字ずつヒントを出します。たとえば `ls` が正解なら、最初は「lから始まる2文字です」と表示します。

### 4.4 慣れてきたら「目的を達成する」

中盤では、明示された手順をそのまま打つのではなく、ミッションを読んで必要なコマンドを選ぶ形式にします。

例:

> `/home/app/config` に移動し、設定ファイルの一覧を確認してください。  
> `app.conf` の中身を確認し、ポート番号が何番か報告してください。

学習者は以下のような操作を自分で選びます。

```bash
cd /home/app/config
ls
cat app.conf
```

### 4.5 高難易度では「事故の予兆」を体験する

後半では、実務に近い緊張感を持たせます。ただし実際のOSには影響を与えず、ゲーム内の仮想ファイルシステムで安全に再現します。

例:

- 本番環境風のディレクトリで不要ファイルを削除する
- 権限を誤って広げるとセキュリティインシデントになる
- 設定ファイルを間違えて変更するとサービスが停止する
- ログを見ずに再起動すると原因不明のまま復旧が遅れる
- `rm -rf`、`chmod 777`、`chown` などの危険操作には確認や警告が出る

このゲームの重要な学びは、コマンドを知ることだけではなく、「何を確認してから実行するべきか」を覚えることです。

## 5. ゲームの基本構成

### 5.1 画面構成

Canvas要素を中心に、できるだけ本物のLinuxデスクトップ、またはLinuxコンソールを触っているような画面を構成します。

- 仮想ターミナル画面
- ミッション指示パネル
- ヒント表示エリア
- 仮想ファイルツリー
- ステータス表示
- インシデントメーター
- 入力履歴
- スコア、達成率、タイム

Canvasを使うことで、ターミナルの演出、画面揺れ、障害アラート、ログの流れ、ミッション達成エフェクトなどを自由に作れます。

ただし、入力フォームやアクセシビリティを考えると、コマンド入力欄だけはHTMLの `input` または `textarea` を重ねる構成が現実的です。

### 5.2 Linux画面と学習UIの分離

ゲーム画面では、Linuxの画面そのものと、問題文やヒントなどの学習UIを視覚的に分けます。

Linux画面は、できるだけ実機に近い黒背景のCLI、プロンプト、ログ、ファイル一覧、エラー表示で表現します。一方で、問題文、ヒント、達成条件、解説は、画面上に浮いている別レイヤーとして表示します。

これにより、学習者は「ここがLinuxの世界」「ここがゲームからのガイド」と直感的に区別できます。

表現例:

- Linux画面: 黒背景、等幅フォント、緑または白の文字、シンプルなプロンプト
- 学習UI: 背後のLinux画面が少しだけ感じられる半透明の浮遊パネル、軽いブラー、読みやすい日本語テキスト
- 警告UI: Linux画面内のエラー表示と、ゲーム側の警告パネルを併用
- 解説UI: ミッション完了後にLinux画面の上へ重なる形で表示

### 5.4 説明画面の体験

遊び方や世界観は、ゲーム内のマニュアルファイルを `vi` で開く形にします。

例:

```bash
vi manual.txt
```

説明画面は、見出しをASCIIアートで表示し、本文は読みやすい日本語テキストにします。説明は番号付きで構成し、初学者が順番に読み進められるようにします。

`vi` は実際の編集機能を持たせず、閲覧専用の簡易ビューアとして実装します。インサートモードはありません。終了は `:q!` または `:wq` で行います。マニュアル本文の末尾にも、戻り方を明記します。

マニュアル表示例:

```text
 _   _                 _____         ____  _
| | | | _____      __ |_   _|__     |  _ \| | __ _ _   _
| |_| |/ _ \ \ /\ / /   | |/ _ \    | |_) | |/ _` | | | |
|  _  | (_) \ V  V /    | | (_) |   |  __/| | (_| | |_| |
|_| |_|\___/ \_/\_/     |_|\___/    |_|   |_|\__,_|\__, |
                                                    |___/

1. コマンドを入力してEnterで実行します。
2. まずは pwd で現在地を確認しましょう。
3. ls でファイルやディレクトリを確認できます。
4. cd でディレクトリを移動できます。
5. 危険なコマンドは、実行前に必ず確認しましょう。

終了するには :q! または :wq を入力してください。
```

この体験により、学習者は「Linuxではファイルを開いて読む」「終了方法を知らないと戻れない」という実務でよくあるつまずきを、安全に体験できます。

### 5.5 基本ゲームループ

1. ミッションが表示される
2. 学習者がコマンドを入力する
3. 仮想Linux環境がコマンドを解釈する
4. 結果がターミナルに表示される
5. ミッション条件が満たされたか判定される
6. 必要に応じてヒント、警告、インシデント演出が出る
7. クリア後に解説と次のミッションが表示される

## 6. コマンド学習ステージ案

### Stage 1: ターミナル入門

目的: コンソール操作に慣れる。

学習コマンド:

- `pwd`
- `ls`
- `cd`
- `clear`
- `history`

ミッション例:

- 現在のディレクトリを確認する
- `documents` ディレクトリに移動する
- ディレクトリ内のファイル一覧を表示する
- 1つ前のディレクトリに戻る

### Stage 2: ファイルの確認

目的: ファイル内容の確認方法を覚える。

学習コマンド:

- `cat`
- `less`
- `head`
- `tail`
- `wc`

ミッション例:

- `readme.txt` の内容を確認する
- `access.log` の末尾10行を確認する
- エラーログから `ERROR` を含む行を探す

### Stage 3: ファイル操作

目的: コピー、移動、削除の基本を覚える。

学習コマンド:

- `touch`
- `mkdir`
- `cp`
- `mv`
- `rm`
- `rmdir`

ミッション例:

- 指定されたディレクトリにバックアップフォルダを作成する
- `app.conf` を `app.conf.bak` としてコピーする
- 不要な一時ファイルだけを削除する

### Stage 4: 権限と所有者

目的: パーミッションの読み方と変更のリスクを覚える。

学習コマンド:

- `ls -l`
- `chmod`
- `chown`
- `groups`
- `id`

ミッション例:

- 実行権限がないスクリプトに適切な権限を付与する
- `chmod 777` がなぜ危険かを体験する
- 設定ファイルを誤って全員書き込み可能にした場合のインシデントを確認する

### Stage 5: 検索と調査

目的: 実務で頻出する調査コマンドを覚える。

学習コマンド:

- `grep`
- `find`
- `sort`
- `uniq`
- `cut`
- `awk`

ミッション例:

- ログから特定IPのアクセスを抽出する
- 500エラーが発生した時刻を調べる
- サイズの大きいログファイルを探す

### Stage 6: 障害対応シミュレーション

目的: コマンドを使って原因調査、復旧判断をする。

学習コマンド:

- `ps`
- `top`
- `df`
- `du`
- `systemctl`
- `journalctl`
- `curl`

ミッション例:

- Webサービスが応答しない原因を調査する
- ディスク使用率が100%になった原因ファイルを探す
- ログを確認して設定ミスを特定する
- サービスを再起動して復旧確認する

## 7. ミッション例

### 初級ミッション: はじめてのディレクトリ移動

目的:

- 現在地を確認する
- 指定ディレクトリへ移動する
- ファイル一覧を見る

指示:

> あなたは研修用サーバーにログインしました。  
> `/home/trainee/projects` に移動して、中にあるファイルを確認してください。

想定コマンド:

```bash
pwd
cd /home/trainee/projects
ls
```

クリア条件:

- 現在地が `/home/trainee/projects` である
- `ls` を実行している

### 中級ミッション: 設定ファイルのバックアップ

目的:

- 設定変更前にバックアップを取る習慣を覚える

指示:

> `/etc/myapp` にある `app.conf` を変更する前に、バックアップを作成してください。  
> バックアップ名は `app.conf.bak` にしてください。

想定コマンド:

```bash
cd /etc/myapp
ls
cp app.conf app.conf.bak
ls
```

クリア条件:

- `/etc/myapp/app.conf.bak` が存在する
- 元ファイル `app.conf` が消えていない

警告条件:

- `mv app.conf app.conf.bak` を実行した場合、元ファイルが消えるため注意メッセージを表示する
- バックアップ前にファイルを書き換えようとした場合、減点する

### 上級ミッション: 権限事故を防ぐ

目的:

- 権限変更のリスクを理解する

指示:

> デプロイスクリプト `deploy.sh` が実行できません。  
> 必要最小限の権限を付与して、実行可能にしてください。

想定コマンド:

```bash
ls -l deploy.sh
chmod 755 deploy.sh
ls -l deploy.sh
```

インシデント条件:

- `chmod 777 deploy.sh` を実行すると、セキュリティ警告が発生する
- `chmod -R 777 /` のような危険コマンドは重大インシデント扱いにする

演出:

- 画面右上のインシデントメーターが上昇
- セキュリティチームからの通知風メッセージ
- クリア後に「なぜ777が危険か」を短く解説

## 8. 体験モック案

最初のモックでは、ブラウザ上で以下を体験できるようにします。

### 画面

- 左側: Canvasで描画したターミナル
- 右側: ミッションカード
- 下部: コマンド入力欄
- 上部: ステージ名、スコア、ヒント残数

### 実装する仮想コマンド

MVPでは以下だけで十分です。

- `pwd`
- `ls`
- `cd`
- `cat`
- `cp`
- `chmod`
- `clear`
- `help`

### 初回モックのミッション

1. `pwd` で現在地を確認する
2. `cd /home/trainee/projects` で移動する
3. `ls` で一覧を確認する
4. `cat readme.txt` で内容を見る
5. `cp app.conf app.conf.bak` でバックアップを作る
6. `chmod 755 deploy.sh` で実行権限を付ける

### 失敗時の反応

- 存在しないディレクトリへ移動すると `No such file or directory`
- `chmod 777` を実行すると警告表示
- `rm` を実行すると、MVPでは「このミッションでは削除は禁止」と表示
- 間違ったコマンドを3回続けると、段階的なヒントを出す

## 9. ゲーム性のアイデア

### 9.1 スコア要素

- 正しいコマンド実行で加点
- ヒント使用で少し減点
- 危険操作で大きく減点
- 事前確認をするとボーナス
- クリア後に安全度ランクを表示

### 9.2 タイピングゲーム要素

- 練習モードでは意味とコマンドを同時に表示する
- 学習者は表示されたコマンドを正確にタイピングする
- 本番モードでは意味だけを表示する
- 本番モードは20問の重複なしランダム出題にする
- 本番モードはタイムアタック形式にする
- Enter時の不正解は1回につき1秒ペナルティにする
- 結果画面で最終タイム、正確率、ランクを表示する

### 9.3 緊張感の演出

- 障害発生時にログが流れる
- サービス監視アラートが表示される
- ミスによりインシデントメーターが上がる
- 正しく復旧するとメーターが下がる
- 重大ミス時にはポストモーテム風の振り返りが出る

### 9.4 学習補助

- コマンドごとの短い説明
- 入力途中のサジェスト
- 1段階目ヒント: 使うコマンド名だけ
- 2段階目ヒント: 引数の考え方
- 3段階目ヒント: ほぼ答え
- クリア後に、実務では何に注意するかを解説
